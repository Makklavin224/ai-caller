================================================================================
  voice-bot-vectra — INSTALL на РФ VPS (Ubuntu 22.04)
  Всё в одном compose + HTTP-прокси для внешних API
================================================================================

Архитектура (после уроков первой итерации):
  - VPS ОБЯЗАТЕЛЬНО в РФ. UIS режет SIP-исходящие с не-РФ IP
    ("Регион ВН запрещает использование SIP").
  - Asterisk 18 (Ubuntu 22.04 контейнер) регается на UIS напрямую.
  - Pipecat-бот (Python 3.12 + Pipecat 1.0) ходит в OpenAI / ElevenLabs /
    Deepgram через HTTP-прокси в зарубежной локации (иначе эти API недоступны
    с РФ IP). Прокси должен поддерживать метод CONNECT (для HTTPS/WSS).
  - Аудио Asterisk ↔ Bot: AudioSocket TCP 127.0.0.1:4444 (host network).

Сверено по документации:
  - https://docs.asterisk.org/Latest_API/API_Documentation/Dialplan_Applications/AudioSocket/
  - https://docs.pipecat.ai/server/services/llm/openai
  - https://docs.pipecat.ai/server/services/tts/elevenlabs
  - https://docs.pipecat.ai/server/services/stt/deepgram
  - https://elevenlabs.io/docs/api-reference/text-to-speech/convert

================================================================================
  ШАГ 0. Пререквизиты (собери перед стартом)
================================================================================

1. РФ VPS Ubuntu 22.04. Провайдеры (любой):
     - Selectel  (selectel.ru)
     - Timeweb   (timeweb.cloud)
     - Beget     (beget.com)
     - FirstByte (firstbyte.ru)
   Конфиг: 2 vCPU / 2 GB RAM / 20 GB SSD — с запасом.
   IP должен быть публичный РФ.

2. Открытые порты на фаерволе провайдера (если он есть):
     22/tcp       SSH
     5060/udp     SIP
     10000-10100/udp  RTP (аудио)

3. SIP-аккаунт UIS (из ЛК → Интеграции → SIP):
     UIS_SIP_HOST     (обычно pbx.uiscom.ru)
     UIS_SIP_USER     (SIP-логин, напр. 0575771)
     UIS_SIP_PASSWORD
     UIS_CALLER_ID    (номер из ЛК UIS, без плюса, напр. 74951234567)

4. API-ключи:
     OPENAI_API_KEY       https://platform.openai.com/api-keys
     ELEVENLABS_API_KEY   https://elevenlabs.io/app/settings/api-keys
     ELEVENLABS_VOICE_ID  https://elevenlabs.io/app/voices (после клонирования)
     DEEPGRAM_API_KEY     https://console.deepgram.com  ($200 бесплатно)

5. HTTP-прокси в зарубежной локации:
     URL формата: http://USER:PASS@HOST:PORT
     Должен поддерживать CONNECT для портов 443.

================================================================================
  ШАГ 4. Заполняем .env
================================================================================

nano /opt/voice-bot-vectra/.env

Минимум для запуска — заполнить ВСЕ поля ниже:

  # === HTTP-прокси (критично для работы внешних API из РФ!) ===
  HTTPS_PROXY=http://USER:PASS@HOST:PORT
  HTTP_PROXY=http://USER:PASS@HOST:PORT
  NO_PROXY=localhost,127.0.0.1

  # === UIS SIP ===
  UIS_SIP_HOST=pbx.uiscom.ru
  UIS_SIP_USER=<твой-SIP-логин>
  UIS_SIP_PASSWORD=<пароль>
  UIS_CALLER_ID=<номер-из-UIS-без-плюса>

  # === OpenAI ===
  OPENAI_API_KEY=sk-...
  OPENAI_BASE_URL=https://api.openai.com/v1
  OPENAI_MODEL=gpt-4.1-mini

  # === ElevenLabs ===
  ELEVENLABS_API_KEY=sk_...
  ELEVENLABS_VOICE_ID=<voice-id-из-ElevenLabs>
  ELEVENLABS_MODEL=eleven_multilingual_v2

  # === Deepgram ===
  DEEPGRAM_API_KEY=...

Проверка, что ничего не забыто:
  grep -E "^[A-Z_]+=$" .env
  (строк быть не должно; если есть — заполни недостающие)

================================================================================
  ШАГ 7. Проверка SIP-регистрации UIS
================================================================================

./scripts/check-trunk.sh

Ждём:
  === PJSIP registrations ===
  uis-reg/sip:pbx.uiscom.ru   uis-auth   Registered

  === PJSIP endpoints ===
  uis-endpoint                             Not in use
  Contact uis-aor/sip:pbx.uiscom.ru   Avail   RTT < 100ms

Если статус Rejected:
  - неверный логин/пароль в .env → исправь и docker compose restart asterisk
  - IP VPS не в whitelist UIS → добавь в ЛК UIS

================================================================================
  ШАГ 8. Sanity-check прокси + внешних API
================================================================================

# Прокси живой?
docker compose exec bot curl -sS -o /dev/null -w "проxy HTTP %{http_code}\n" \
  --max-time 10 https://httpbin.org/ip
# ожидаем HTTP 200 (и в теле будет IP твоего прокси, а не РФ IP VPS)

# OpenAI?
docker compose exec bot bash -c 'curl -sS -o /dev/null -w "OpenAI HTTP %{http_code}\n" \
  --max-time 10 https://api.openai.com/v1/models \
  -H "Authorization: Bearer $OPENAI_API_KEY"'
# ожидаем HTTP 200

# ElevenLabs?
docker compose exec bot bash -c 'curl -sS -o /dev/null -w "ElevenLabs HTTP %{http_code}\n" \
  --max-time 10 https://api.elevenlabs.io/v1/voices \
  -H "xi-api-key: $ELEVENLABS_API_KEY"'
# ожидаем HTTP 200

# Deepgram?
docker compose exec bot bash -c 'curl -sS -o /dev/null -w "Deepgram HTTP %{http_code}\n" \
  --max-time 10 https://api.deepgram.com/v1/projects \
  -H "Authorization: Token $DEEPGRAM_API_KEY"'
# ожидаем HTTP 200

Если HTTP 000 / timeout → прокси не работает (неверный USER:PASS, порт,
или прокси не поддерживает CONNECT). Проверь:
  docker compose exec bot bash -c 'echo $HTTPS_PROXY'

================================================================================
  ШАГ 9. Тестовый звонок на свой мобильный
================================================================================

# Терминал 1 — логи бота
docker compose logs -f bot

# Терминал 2 — логи Asterisk (SIP-сигналинг)
docker compose logs -f asterisk

# Терминал 3 — звонок (формат: 7XXXXXXXXXX, без плюса)
./scripts/originate.sh 79XXXXXXXXX

Что должно произойти:
  1. Asterisk отправит INVITE в UIS (→ Called 7...@uis-endpoint)
  2. UIS ответит progress + ring → твой мобильный зазвонит
  3. Снимаешь трубку → Asterisk: AudioSocket(UUID,127.0.0.1:4444)
  4. Бот: "AudioSocket connection from ... Starting pipeline"
  5. Анна по голосу ElevenLabs:
     "Алло, здравствуйте! Это Анна, компания ВЕктра..."
  6. Отвечаешь — Deepgram распознаёт (streaming) → OpenAI отвечает →
     ElevenLabs озвучивает, и так по кругу до финальной реплики.

================================================================================
  ОТЛАДКА
================================================================================

A. Asterisk не регается в UIS:
   docker compose exec asterisk asterisk -rx "pjsip show registrations"
   docker compose exec asterisk asterisk -rx "pjsip show transports"
   - Rejected → user/pass неверны
   - Unregistered → сеть/фаервол/whitelist UIS

B. UIS: "Регион ВН запрещает использование SIP":
   - IP твоего VPS не РФ. Пересаживайся на РФ-провайдера.

C. Бот крашится при старте:
   docker compose logs bot
   - Чаще всего: нет одного из API-ключей
   - ModuleNotFoundError → API Pipecat 1.x слегка разошёлся → сообщай,
     поправим (пока сверено на pipecat-ai 1.0.0)

D. Прокси не отвечает:
   docker compose exec bot curl -v -x $HTTPS_PROXY https://api.openai.com/v1/models
   - 407 Proxy Authentication Required → USER:PASS неверны
   - timeout → HOST:PORT неверны / прокси не работает
   - если в USER/PASS есть спецсимволы (@/:) — URL-encode их:
       @ → %40   : → %3A   / → %2F

E. "making progress" но звонок до клиента не доходит:
   - в ЛК UIS: баланс, CALLERID, разрешены ли исходящие
   - не тот UIS_CALLER_ID в .env → поправь + restart
   - номер куда звоним в неподдерживаемом регионе UIS

F. Звук идёт, но булькает / прерывается:
   - RTT до UIS (check-trunk.sh) должен быть < 100ms
   - docker compose exec bot ping -c 3 <proxy-host> — < 200ms
   - если больше 300ms — смени прокси на ближе к РФ (Финляндия, Германия)

G. Бот слышит, но отвечает с задержкой > 2 сек:
   - Проверь `docker compose logs bot | grep -i latency`
   - Основные задержки: OpenAI first-token ~500мс, ElevenLabs TTS ~300мс
   - Если сильно больше — проблема в прокси

Когда single-call работает стабильно:
  - простой цикл:
      while read n; do ./scripts/originate.sh "$n"; sleep 8; done < numbers.txt
  - либо Vicidial поверх того же Asterisk
  - либо собственный диалер через AMI с контролем одновременных каналов
    (UIS обычно даёт 5-20 одновременных SIP-каналов на тарифе)

================================================================================
  БЕЗОПАСНОСТЬ
================================================================================

  - .env содержит секреты; он в .gitignore, в репу не коммитится.
  - При смене прокси/ключей → nano .env + docker compose restart bot.
  - Если твой proxy USER:PASS утёк (лог, screenshot и т.п.) — ротируй у
    поставщика прокси и меняй в .env.
  - SIP-пароль UIS не должен попадать в логи Asterisk в продакшене
    (поставь level=notice в logger.conf).
